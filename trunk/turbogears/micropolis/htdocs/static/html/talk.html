Outside the Box: Micropolis Eduverse

Vision

  • Micropolis, the open source version of SimCity, is a fun, engaging game, that helps teach kids about science, economics, language, mathematics, art and politics.
  • Its goal is to fulfill SimCity's potential as a microworld for children's learning and exploration.

What is this running in the browser?

  • Started with the originl Mac version of SimCity Classic.
  • Ported it to Unix, using the NeWS window system, writing the UI in PostScript.
  • Ported it to Unix, using the X11 window system, writing the UI in TCL/Tk.
  • Designed and implemented colaborative multi player support.
  • Ported and adapted it to the OLPC, so EA could release it under GPLv3.
  • Stripped it down, removed the UI and scripting language, translated it to C++, cleaned it up and refactored it.
  • Integrated it with Python via SWIG.
  • Implemented C++ tile engine using the Cairo graphics library, and integrated it with Python via C++.
  • Wrote GTK based user interface in Python. Will support Sugar, but now runs on Windows/Mac/Linux.
  • Wrote a web server based application. Python/TurboGears/SQLAlchemy/Genshi/Cairo/Ming.
  • Implementing Flash based user interface to web server based application.
  • Using OpenLaszlo, to support both DHTML/JavaScript in ordinary web browsers, as well as Flash.
  • Goal is to develop it into an educational multi player game, community sharing and blogging site, etc.

How can we apply Micropolis educationally?

"There is no win or lose, just cause and effect." -Jeff Braun, Maxis
  • Motivating students to think about complex systems, cause and effect, responsibility and consequences.
  • Science
  • Social Studies
  • Math
    • Export spreadsheets of data from the simulation, so students can perform experiments and analyze the results in a spreadsheet or with other tools.
  • Environmentalism
    • Pollution. Global warming. Transportation. Energy policy.
  • Writing
    • The Sims Exchange.
    • Storytelling.
    • Blogging.
    • Write newspaper articles about things happening in the city.
    • Events in the game trigger opportunities to write newspaper stories.
    • Blogging dialog pops up, with tools for editing the story, including inserting illustrations.
    • Snapshots of the city, graphs, overlays, information visualizations, evaluations).
    • Stories link to save file snapshots, and are published on the web sites as a newspaper or blug, so other players can browse and play them.
  • Economics
    • Personal finance.
    • Responsible financial planning.
    • Borrowing and paying back money. Investment. Insurance.
    • Take out a loan. Invest in infrastructure. Pay the loan off over time.
    • Show graphs of cash flow, compounded interest, payments, etc.
    • Build up and maintain a credit rating, so you can borrow more money in the future.
    • Save points so you can rewind to decision points and play "what-if" games.
  • Politics
    • Democratic process.
    • Debating issues.
    • Polder model.
      • Consensus decision making policy.
      • Cooperation despite differences.
      • Set aside differences for a greater purpose.
    • Role playing, representing different interests.
    • Writing proposals.
    • Voting on other players' proposals.
      • A proposal can consist of a title, a description, some illustrations taken from the game or uploaded, bookmarks on the map, links to web sites related to the issue, an overlay of graphics and text drawn on the map, a budget requirement, some hypothetical map editing commands (build a zone, bulldoze and area, change the tax rate, etc), a message board for discussing the issue, a time limit until the election, and a voting interface for deciding the issue.
      • Once the issue is decided, the editing commands are applied (if possible), or the player who proposed it is given the authority to implement the proposal (given $1000 to use on building roads anywhere they decide, etc.)
      • Hand Monderman's shared space.
      • Colaborative. Face to face. Looking peers in the eye. Civic responsability. Mixed use. Multiple roles. Slow network. Slow down the game to a more thoughtful pace. Endless discussion. Gridlock. Polder model.
      • "The reasoning behind the idea is that it will result in improved road safety by forcing users to negotiate their way through shared areas at appropriate speeds and with due consideration for the other users of the space."
      • A reason for the apparent paradox that reduced regulation leads to safer roads may be found by studying the risk compensation effect. Shared Space describe the effect:
      • Shared Space is successful because the perception of risk may be a means or even a prerequisite for increasing objective safety. Because when a situation feels unsafe, people are more alert and there are fewer accidents.
  • Programming
    • Open up the simulation so students can read and understand the source code, modify and extend the way the simulation works.
    • Visual programming languages, like eToys, Robot Odyssey, SimAntics, etc.
    • Provide a well factored, modular "soup to nuts" open source programming example that other people can extend or use as a starting point for their own projects.
